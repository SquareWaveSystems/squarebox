# Domain docs

Squarebox is a single-context repository.

Before changing behavior:

- Read `CONTEXT.md` and use its terms in code, issues, tests, and documentation.
- Read relevant decisions in `docs/adr/`.
- Treat an absent document as permission to proceed, not as a blocker.
- Surface any proposed change that contradicts an accepted ADR.

Add a term only when it is specific to Squarebox and resolves real ambiguity. Keep definitions short and record hard-to-reverse, surprising trade-offs as concise ADRs.
