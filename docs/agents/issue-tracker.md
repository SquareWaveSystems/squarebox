# Issue tracker: GitHub

Issues and PRDs for this repository live in GitHub Issues. Use the `gh` CLI from this clone so the repository is inferred from `origin`.

## Conventions

- Create issues with a concise outcome-oriented title and a body containing evidence, acceptance criteria, and dependencies.
- Read the issue body, labels, and comments before implementation.
- Use `gh issue list` with JSON output for triage and planning.
- Comment with verification evidence before closing an issue.
- Apply the role labels mapped in `docs/agents/triage-labels.md`.
- Do not close an issue merely because a patch exists; close it when its acceptance criteria pass.
